1. Модульный тест - это тест, которые проверяет какую-то определенную единицу кода (будь то метод, класс или что-то
 большее). Проверяемая единица кода может быть связана с другими единицами кода, которые мы не проверяем, т.е. обладать
 зависимостями. Если при проверке такой единицы кода со связями будет ошибка, то непонятно, где она конкретно: в самом
 проверяемом коде или в связанных с ним объектах. Поэтому одним из способов избавиться от ненужных зависимостей (кусков кода,
 которые не тестируешь) - это заглушки (фиктивные зависимости).

 2. Для того, чтобы проверить, что метод был вызван с определенными аргументами, можно использовать dummy - заглушки. Они не имеют
 никакой логики внутри, а только создают объект нужного типа для передачи его в качестве аргумента.

 3. Для теста, в котором просто нужно вернуть из метода определенное значение или исключение, можно использовать stub -
 заглушки. В таких объектах также нет никакой реализации, а только возвращаются предопределенные значения.

 4. Для реализации теста, который порверяет способность взаимодействия с б.д. или снешним api, можно исползовать заглушку
 типа fake. Внутри такой заглушки можно создать простую коллекцию для хранения данных и логику.
